   - Admin sponsor page now includes Hide Open Sponsorships button.

6. Snapshot export upgraded
   - /admin/export-snapshot.zip now includes attendees, field metrics, sponsorship catalog, sponsorship sales, and show sponsors if those tables exist.

Deploy steps:
1. Back up your current project folder and Railway database volume first.
2. Copy the files above into the matching locations.
3. Commit and push:
   git add app.py database.py sponsorship_blueprint.py templates/admin_shows.html templates/placeholder_claim.html templates/admin_sponsors.html templates/sponsorship.html templates/admin_show_mode.html

5. Open /admin/show-mode during the event.
6. Use /admin/export-snapshot.zip or Close Voting and Export when wrapping up.

EXPORT UPGRADE v4 — 2026-05-04

New export behavior:
1. Manage Shows now has a Download Data ZIP button for every show, active or past.
2. Manage Shows now has a Close + Export button for every show.
3. Set Past now automatically saves a show export ZIP before returning to Manage Shows.
4. Saved export ZIPs are stored on Railway at:
   /data/exports
5. The Manage Shows page displays the most recent saved export ZIPs with download links.

New routes:
- GET  /admin/shows/<show_id>/export.zip
  Exports that show by ID. It does not require the show to be active.

- POST /admin/shows/<show_id>/close-and-export
  Closes voting, marks the show past, saves the export ZIP to /data/exports, and downloads it.

- GET /admin/exports/<filename>
  Downloads a ZIP that was already saved in /data/exports.

Legacy routes kept:
- /admin/export-snapshot.zip still exports the active show if one exists, but redirects to Manage Shows with a message if no show is active.
- /admin/close-voting-and-export still works for the active show and now saves the ZIP permanently too.

After deploying:
1. Go to /admin/shows.
2. For the old show, click Download Data ZIP to get the current data.
3. Use Close + Export at the end of future shows.
4. If you click Set Past, an export is automatically saved to Railway /data/exports.
