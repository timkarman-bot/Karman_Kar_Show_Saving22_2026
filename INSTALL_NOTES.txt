Karman Kar Shows & Events — Show-Specific Registration Upgrade
Date: 2026-05-07

Purpose
- Allows multiple upcoming shows to take registrations at the same time.
- Adds show-specific registration URLs:
  /register/<show-slug>
- Keeps /register as a safe fallback to the current active show.
- Updates the show detail page and registration form so people register for the intended show, not whichever show is active.

Files included
- app.py
- database.py
- templates/register.html
- templates/show.html
- templates/admin.html
- templates/admin_shows.html
- templates/placeholder_claim.html

Important URL examples
- /register/fathers-day-cruise-in-2026
- /show/fathers-day-cruise-in-2026

Post-install checks
1. Run:
   python -m py_compile app.py database.py sponsorship_blueprint.py

2. Confirm routes exist:
   findstr /n "register/<show_slug>" app.py
   findstr /n "register_page_for_show" app.py templates\show.html
   findstr /n "register_submit_for_show" app.py templates\register.html

3. Test in browser:
   /show/<show-slug>
   /register/<show-slug>
   /register

Deployment note
- Use show-specific registration links on flyers, QR codes, emails, and social media.
- The plain /register route should only be treated as an admin convenience/fallback.
